# API Automation Framework Workflow

## Directory Map

- `api_auto_test/common`: framework core code, including request sending, assertions, extraction, context replacement, logging, YAML loading, and Allure attachments.
- `api_auto_test/config`: environment, base URL, headers, timeout, report, and other shared settings.
- `api_auto_test/data`: YAML test data. Current bundled examples include topic and index dictionary modules.
- `api_auto_test/scripts`: helper scripts for running tests or report generation.
- `api_auto_test/tests`: pytest test entry files grouped by module.
- `api_auto_test/debug_one_case.py`: debug a selected YAML case directly.
- `api_auto_test/conftest.py`: pytest fixtures and run setup.
- `api_auto_test/pytest.ini`: pytest configuration.
- `run.py`: project-level runner for executing pytest and generating reports.
- `.vscode`: editor launch/settings examples if present in the source project.

## YAML Case Pattern

Each YAML file should represent one business module. Prefer this shape:

```yaml
module: 模块中文名
cases:
  - name: 01_创建主题
    skip: false
    skip_reason: ""
    request:
      method: POST
      url: /api/example
      headers: {}
      json: {}
      params: {}
    extract:
      domain_name: $.data.name
    asserts:
      - type: eq
        actual: $.code
        expected: 0
        message: 状态码断言
```

If a case should be skipped, set:

```yaml
skip: true
skip_reason: 跳过原因
```

## Context Correlation

Use extraction to save values from earlier responses, then reference them in later requests or assertions.

Common pattern:

```yaml
extract:
  domain_name: $.data.records[0].domainName
```

Later cases can reference:

```yaml
json:
  domainName: ${domain_name}
```

If context is missing, improve the error by showing the case name, parameter name, and the missing variable.

## Assertion Output

Keep assertion output readable in Chinese. Prefer messages like:

- `断言：预期结果=...，实际结果=...`
- `提取：变量名=...，JSONPath=...`

Attach or log enough request/response detail to distinguish:

- module name;
- case name;
- method and URL;
- request headers/body/params;
- response status and body;
- assertion expected value and actual value;
- extraction variable and extracted value.

## Adding A New Module

1. Create a new YAML file under `api_auto_test/data`, for example `new_module.yaml`.
2. Set `module` to the Chinese module name that should appear in Allure.
3. Add one or more cases under `cases`.
4. Create or reuse a pytest entry under `api_auto_test/tests`.
5. Point the test file to the new YAML data file.
6. Run one case with `debug_one_case.py`, then run the full suite with `run.py`.

## Running

Typical commands from the project root:

```powershell
python run.py
pytest -s -v
pytest -s -v api_auto_test/tests
```

Allure result files should be generated into the framework report directory. The HTML report is generated by the runner when Allure CLI is available.

## Debugging One Case

Use `api_auto_test/debug_one_case.py` for direct single-case debugging. Edit the target YAML filename and case name/index in that file, then run it in Python or an IDE debug session.

If a later query/edit/delete case depends on values created by an earlier case, debug the create case first or make the debug script run the dependency case before the selected case.

## Converter Usage

Use the converter in `assets/converter` when the user provides copied cURL/request-response data and wants YAML cases. The generated YAML should still be checked for:

- valid JSON body;
- long bodies not truncated by Excel;
- correct module name;
- useful case name;
- required extracts;
- Chinese assertion messages.

## Refreshing This Skill

When refreshing the bundled template from the user's working project:

1. Copy the current framework into `assets/framework-template`.
2. Include config, data, tests, common, scripts, debug files, pytest config, runner, and converter.
3. Exclude runtime output:
   - `reports`
   - `logs`
   - `data/runtime_context.json`
   - `__pycache__`
   - `.pytest_cache`
4. Package the skill as a flat zip whose root contains `SKILL.md`, `assets`, `references`, and optional `agents`.
